Typora使用PicGo-Core插件配置图床

https://blog.csdn.net/to_free/article/details/114981807

在编辑md文件时，经常需要插图片。如果将图片保存在服务器的本地中，会占空间、拖慢速度，而图床工具，是一种自动把本地图片转换成链接的工具，Picgo是一种知名的图床工具。我们使用“PicGo-Core(command line)+gitee仓库”方案，免费哦。

著名的的md编辑器Typora支持PicGo的插件，只需在Typora里下载即可，使用体验更好，且避免了一些使用PicGo+github软件的坑。

步骤如下：

  • 安装Typora

  • （可改语言为中文）文件-偏好设置-通用

  • 在Typora中下载PicGo-Core(command line)插件（不用下载PicGo

    • 打开Typora，依次点击文件-偏好设置-图像；

    • 按下图所示勾选插入图片时的选项，并在上传服务设定选择PicGo-Core(command line)，点击下载或更新按钮下载插件。

  • 借助picgo安装gitee图床插件

    • 找到上一步中下载的picgo二进制文件，在此只考虑windows系统，一般在C:\Users\用户名\AppData\Roaming\Typora\picgo\win64\文件夹，文件名为picgo.exe

      提示：若未找到AppData，则在导航栏的查看里勾选隐藏的项目。

    • 打开命令行进入当前文件夹的位置，运行.\picgo.exe install gitee-uploader。

  • 注册/登录gitee，并创建一个仓库

    注意红框框起来的部分。如果选择私有，那么无法通过网址访问图片；如果没有readme，后面验证图片上传会报错。其他不用考虑。

  • 通过设置-私人令牌申请一个私人令牌（可以申好多个），记得保存。

    其他的权限选了也可以。

    点击打开配置文件

  • 点验证图片上传选项

    等几秒会出现绿色的成功提示，这时在你的仓库中的某文件夹中可以看到有图片上传了。

  • 接下来就可以使用了

    当你在Typora中插入了本地链接的图片，那么在该链接上右击，会出现上传图片的选项，点它，就上传了，可以在仓库里看到了。
