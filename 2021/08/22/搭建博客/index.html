wordpress建站遇到的坑（都是镜像的锅qwq）

发表于2021-08-22|更新于2021-08-29
|字数总计:1.1k|阅读时长:3分钟|阅读量:

一、购买域名、云服务器

可以都在阿里云上买。服务器在国内需要备案（需要等20天左右），在香港或国外不需要备案，否则不能访问，是否需要备案和域名无关，只和服务器的地方有关。如果不想备案，在各平台上买服务器时选择香港或国外就行。我是在阿里云上买的，1核2G，选择镜像市场里的Wordpress平台（CentOS8.2 LNMP PHP7.4）镜像。

（LNMP 是 Linux、Nginx、MySQL 和 PHP 的缩写，是 WordPress 博客系统依赖的基础运行环境。）

二、配置服务器

1. 阿里云-域名-解析-添加记录

记录值填公网ip

按理说主机记录只填一条填*表示www和@都行，但是我后面遇到，服务器里ping不到@，改成两条就好了。

2. 远程连接服务器

现在访问域名或主机ip应该是404

进来后竟然是这样的，没有图形界面。

3. 配置环境

发现ReadMe

在线文档

按照简介里的链接开放安全组端口

这个镜像里数据库已经给你建好了叫wordpress，1.2里有个LNMP环境镜像使用手册

参考这个手册里的1.13.1 如何部署一个PHP网站

数据库不用新建，按教程改一下密码就好
按教程添加虚拟主机

虚拟主机域名填网站域名，不仅需要域名解析到服务器，服务器也要连接到域名

在这里我出现了错误

后来发现，我重启了服务器，它的公网ip变了，就不对了。这个问题解决的挺玄学的。

可以如上图的底部所示ping你的域名和ip，看看是否正常。

我也跟着网上别人的解决方式装了nscd，挺有用的，改了域名解析的主机地址没有更新时，就可以用这个清除缓存。

总之，我连成功了

按教程创建FTP账号

不知道实际上有没有用

按教程做1.13.1剩下的内容

注意，wget的网址是https://cn.wordpress.org/latest-zh_CN.tar.gz

他那个不是最新的

  • 装完，这里我又遇到了问题，我的主机ip和域名都啥也访问不到（其实从头到现在就没访问到过）

按照那个文档里1.5.3的方法二：phpMyAdmin登录到这个网站里，我之前一直访问不到，后面不知怎么就能访问到了。

  • 这里用户密码错误也是个问题。首先看看能否登录进数据库

    或许需要启动数据库service mysqld start（报redirect的warning没关系，它实际上执行了）

    命令行里输入mysql -uroot –p

    可能报错Access denied for user 'root'@'localhost' (using password: NO)

    我按https://www.fujieace.com/linux/1045.html流程走了一下（改密码那一步没做）就好了

  • 现在，网址栏里输入主机ip可能还是显示建立数据库出错，但是服务器里连接是正常的

    那么下面就修改配置文件wp-config.php

    尽管根目录下只有两个东西，但是输入cd /data就可以进入另一番天地

    你也可以使用命令find / -name wp-config.php找这个文件，记得加这个杠，不然搜不到

    （用vim命令）进去把define( 'DB_PASSWORD', 'd3gHidejkJG' );改成你的密码

    你看这个要不要改呢define( 'DB_HOST', 'localhost' );

    不用！服务器的数据库host地址就是本机，热知识：127.0.0.1就是localhost

总之，ip地址能进去了！！泪目

  • 填信息途中用户名和密码是啥？

    按这个解决https://jingyan.baidu.com/article/414eccf64d3ebc6b431f0acb.html

  • 但它咋是这样全是文字啊，啊，是css丢失了。

    登录phpMyAdmin，找到存WordPress的数据库，wp_options表，将siteurl和home的option_value改成公网ip

芜湖，好啦！

  • 等等，ip能访问到，域名访问不到，只有head，其他是空白的，而且也不报错

在http://主机ip/phpMyAdmin中把wp_options表中的home改成域名，siteurl保持为主机公网ip。就好了。这样域名是给别人访问的，只能查看，ip我们自己访问，都在这里建设

下次还是不要这么麻烦了qwq
